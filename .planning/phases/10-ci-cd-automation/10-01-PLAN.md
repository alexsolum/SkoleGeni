  <action>
Create the directory `.github/workflows/` and write the file `deploy-optimizer.yml` with the following exact structure:

**Workflow name:** `Deploy Optimizer`

**Trigger:**
```yaml
on:
  push:
    branches:
      - main
    paths:
      - 'api/**'
      - 'docker/optimizer.Dockerfile'
```
The `branches` and `paths` are AND-ed by GitHub Actions — only pushes to `main` that touch at least one of these paths will trigger the workflow. This satisfies the success criterion that non-optimizer pushes do NOT trigger a deploy.

**Job: `deploy`**
- `runs-on: ubuntu-latest`
- Permissions (MUST be at job level, not just workflow level):
  ```yaml
  permissions:
    contents: 'read'
    id-token: 'write'
  ```
  The `id-token: write` permission is REQUIRED for the OIDC token exchange with WIF. Without it, the auth step fails with a cryptic error.

**Steps (in order):**

1. **Checkout** — `actions/checkout@v4`

2. **Authenticate to GCP** — `google-github-actions/auth@v3`
   ```yaml
   with:
     project_id: '${{ vars.GCP_PROJECT_ID }}'
     workload_identity_provider: '${{ vars.WIF_PROVIDER }}'
     service_account: '${{ vars.WIF_SERVICE_ACCOUNT }}'
   ```
   Uses GitHub Actions variables (not secrets) for project ID and WIF resource names — these are not sensitive and being visible in logs aids debugging.

3. **Configure Docker for Artifact Registry**
   ```yaml
   - name: Configure Docker for Artifact Registry
     run: gcloud auth configure-docker europe-west1-docker.pkg.dev --quiet
   ```
   MUST specify `europe-west1-docker.pkg.dev` explicitly — without the region, only `gcr.io` is configured.

4. **Build and push image**
   ```yaml
   - name: Build and push image
     env:
       IMAGE: europe-west1-docker.pkg.dev/${{ vars.GCP_PROJECT_ID }}/skolegeni-optimizer/optimizer:${{ github.sha }}
     run: |
       docker build -f docker/optimizer.Dockerfile -t "$IMAGE" .
       docker push "$IMAGE"
   ```
   Uses `${{ github.sha }}` as the image tag for traceability and rollback. MUST use `-f docker/optimizer.Dockerfile` because the Dockerfile is not at the default `./Dockerfile` path.

5. **Deploy to Cloud Run** — `google-github-actions/deploy-cloudrun@v3`
   ```yaml
   - uses: google-github-actions/deploy-cloudrun@v3
     with:
       service: 'optimizer'
       region: 'europe-west1'
       image: 'europe-west1-docker.pkg.dev/${{ vars.GCP_PROJECT_ID }}/skolegeni-optimizer/optimizer:${{ github.sha }}'
       flags: >-
         --memory=2Gi
         --timeout=300s
         --concurrency=1
         --max-instances=3
         --allow-unauthenticated
         --set-env-vars=SUPABASE_URL=${{ secrets.SUPABASE_URL }},SUPABASE_ANON_KEY=${{ secrets.SUPABASE_ANON_KEY }}
   ```
   The `>-` YAML block scalar folds the flags into a single space-separated string. All runtime config is declarative in this file — no drift from GCP console changes. The `--allow-unauthenticated` flag MUST be re-applied on every deploy because Cloud Run does not persist IAM policies set by flags.

6. **Verify deployment health**
   ```yaml
   - name: Verify deployment health
     run: |
       sleep 10
       STATUS=$(curl -s -o /dev/null -w "%{http_code}" https://optimizer-ek4bkd34ja-ew.a.run.app/health)
       if [ "$STATUS" != "200" ]; then
         echo "Health check failed: HTTP $STATUS"
         exit 1
       fi
       echo "Health check passed: HTTP $STATUS"
   ```
   The `sleep 10` allows the new revision to become the serving revision after `gcloud run deploy` returns.

**Do NOT:**
- Store any service account key JSON — WIF is mandatory per INFRA-03
- Tag images as `latest` only — always use git SHA
- Use `dorny/paths-filter` — native `on.push.paths` is sufficient
- Use `google-github-actions/setup-gcloud` — the auth action already sets up gcloud
