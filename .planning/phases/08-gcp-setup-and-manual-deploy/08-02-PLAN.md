<task type="auto">
  <name>Task 1: Deploy Optimizer to Cloud Run</name>
  <files></files>
  <action>
    Deploy the timestamped image from Artifact Registry to Google Cloud Run.
    
    Using 2GiB memory (as per research recommendation for OR-Tools) instead of the initial 512Mi request to ensure stability under load.
    
    Command:
    ```bash
    gcloud run deploy optimizer \
      --image europe-west1-docker.pkg.dev/[GCP_PROJECT_ID]/skolegeni-optimizer/optimizer:[TAG_FROM_PLAN_01] \
      --region europe-west1 \
      --project=[GCP_PROJECT_ID] \
      --memory 2Gi \
      --cpu 1 \
      --timeout 300s \
      --concurrency 1 \
      --allow-unauthenticated \
      --cpu-boost \
      --startup-probe-path=/health \
      --liveness-probe-path=/health \
      --set-env-vars "SUPABASE_URL=[SUPABASE_URL],SUPABASE_ANON_KEY=[SUPABASE_ANON_KEY]"
    ```
    
    Note the Service URL from the output.
  </action>
  <verify>
    <automated>gcloud run services describe optimizer --region=europe-west1 --project=[GCP_PROJECT_ID] --format="json(spec.template.spec.containers[0].resources.limits.memory,spec.template.spec.timeoutSeconds)"</automated>
  </verify>
  <done>Cloud Run service 'optimizer' is deployed with 2GiB memory and 300s timeout.</done>
</task>

  <what-built>A live Cloud Run optimizer service.</what-built>

    Perform an authenticated optimization request to the remote service using curl.
    
    1. Obtain the Service URL:
       `URL=$(gcloud run services describe optimizer --region europe-west1 --project=[GCP_PROJECT_ID] --format="value(status.url)")`

    3. Run the smoke test:

       (Ensure the [TEST_PROJECT_ID] exists and contains pupils/constraints)
    
    4. Verify the response contains "classes" and "score".
